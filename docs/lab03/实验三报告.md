### 测试用例

 

| 用例编号         |          |          |          |          |
| ---------------- | -------- | -------- | -------- | -------- |
| 待测试类及方法   |          |          |          |          |
| 测试类及方法     |          |          |          |          |
| 前提条件（如有） |          |          |          |          |
| 用例描述         | 测试步骤 | 期望结果 | 实际输出 | 测试结果 |

*用例编号：唯一标识测试用例的序号，一般是数字或模块名首字母大写+数字序号。*

*待测试类及方法：该用例所测试的类名和方法名*

*测试类及方法：相应的测试代码的类名和方法名*

*前提条件（如有）：执行该测试用例的前提条件，比如碰撞检测，需已创建英雄机和敌机（或道具）。*

*用例描述：用一句话简单总结该测试用例的用意和目的。*

*测试步骤：详细完整的操作过程描述。*

*期望结果：正常情况下的响应结果。*

*实际结果：程序通过测试步骤后实际的响应结果。*

*测试结果：通过或失败*
